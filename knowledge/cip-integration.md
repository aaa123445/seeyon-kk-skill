# CIP 集成平台整理

## 第三方应用接入

CIP 第三方应用一般先做产品登记，再做应用注册。应用注册成功会自动生成应用编号，该编号是后续关联应用、菜单、待办、事件等配置的关键字段。

接入方式包括：

- PC。
- 移动 URL 接入应用（M3/微协同）。
- 本地 H5 应用。
- 本地原生（M3 原生唤醒）。
- PC & 移动 URL。
- PC & 本地 H5。
- PC & 本地原生。

应用注册后，在“应用接入”中双击或点击接入设置继续配置。

## 应用菜单绑定

- 应用菜单仅支持配置了单点登录的注册应用。
- 接入类型需要是 PC 或包含 PC 的混合类型。
- PC 图标从图标库选择，也支持自定义上传。
- 菜单配置要准备应用、菜单名称、图标、URL/入口、排序、权限等。

## 事件绑定

CIP 内置事件可绑定 URL 或接口：

- 类别包括组织模型事件、新闻公告事件等。
- 可设置触发条件；字符串支持 `==`、包含、不包含；数字支持 `>`、`>=`、`<`、`<=`、`==`。
- 处理类型为 URL 时，事件参数以 JSON 调用 URL。
- 处理类型为接口时，通过接口库匹配并进行事件关联数据/参数赋值。

新增人员事件参数示意：

```json
{
  "id": "1211023486123131711",
  "name": "张",
  "memberLoginName": "zhangz",
  "telNumber": "13316167890",
  "code": "009",
  "memberOrgDepartmentName": "部门",
  "orgAccountName": "aa",
  "orgPostId": "-1696775289751451665",
  "orgPostCode": "",
  "orgPostName": "岗位"
}
```

## 组织同步：接口同步

CIP 支持基于第三方接口的组织同步。第三方需要提供符合 CIP 数据规范的固定组织模型：

1. 集团。
2. 单位。
3. 部门。
4. 岗位。
5. 职务。
6. 人员基础信息。
7. 人员任职信息。
8. 人员跨部门兼职信息。

操作路线：

1. 将第三方接口注册到“CIP 集成平台 → 集成资源库 → 第三方应用接口库(V5接口)”。
2. 同步方案选择第三方应用。
3. 同步方式选择接口同步。
4. 点击同步接口绑定。
5. 分别绑定集团、单位、部门、岗位、职务、人员基础、人员任职、人员兼职模型。

约束：

- 最少必须提供 2 个接口：部门接口和人员接口；此时人员接口必须提供任职信息。
- 接口返回结构化数据：JSON / XML / JCOTable。
- 接口返回数据必须体现组织对象归属关系（外键关系）。
- 部门必须是单维度树形结构，不允许同一部门多个上级。
- 主岗唯一，接口必须标明人员唯一主岗部门。
- 返回数据支持时间戳以实现增量同步。
- 不允许只有兼职没有主岗。

## 统一认证 / SSO

文档目录包含 CAS 认证、景安统一认证等入口，但具体接入参数需回查现场文档/版本。做菜单绑定、第三方应用接入前，要先确认 SSO 方式，否则菜单可见但无法免登录穿透。
